Lab 1 AWS SAM — Full Serverless App: Lambda + API Gateway + DynamoDB

SAM (Serverless Application Model) is a CloudFormation extension that simplifies serverless deployments. It appears frequently on the DVA-C02 exam. This lab covers the full lifecycle: write → test locally → deploy → traffic shift.

🏗️ Architecture

📋 SAM Template (template.yaml)

AWSTemplateFormatVersion: '2010-09-09'
Transform: 'AWS::Serverless-2016-10-31'   # ← REQUIRED: marks this as SAM

Globals:                                   # Applied to all functions
  Function:
    Timeout: 30
    Runtime: java17
    MemorySize: 512
    Environment:
      Variables:
        TABLE_NAME: !Ref OrdersTable
        JAVA_TOOL_OPTIONS: "-XX:+TieredCompilation -XX:TieredStopAtLevel=1"

Resources:

  # Lambda Function with API Gateway trigger
  OrdersFunction:
    Type: AWS::Serverless::Function        # SAM resource type
    Properties:
      Handler: com.myapp.OrderHandler::handleRequest
      CodeUri: target/orders-1.0.jar
      Policies:
        - DynamoDBCrudPolicy:              # SAM policy template (shorthand)
            TableName: !Ref OrdersTable
      AutoPublishAlias: live               # Auto-publish version + create alias
      DeploymentPreference:               # CodeDeploy traffic shifting
        Type: Canary10Percent5Minutes     # 10% for 5 min, then 100%
        Alarms:
          - !Ref OrdersFunctionErrorAlarm
      Events:
        GetOrders:
          Type: Api
          Properties:
            Path: /orders
            Method: GET
        CreateOrder:
          Type: Api
          Properties:
            Path: /orders
            Method: POST
        # S3 trigger
        ProcessUpload:
          Type: S3
          Properties:
            Bucket: !Ref UploadBucket
            Events: s3:ObjectCreated:*
            Filter:
              S3Key:
                Rules:
                  - Name: suffix
                    Value: .jpg

  # DynamoDB Table
  OrdersTable:
    Type: AWS::Serverless::SimpleTable
    Properties:
      PrimaryKey:
        Name: orderId
        Type: String
      BillingMode: PAY_PER_REQUEST

  # S3 Bucket for uploads
  UploadBucket:
    Type: AWS::S3::Bucket

  # CloudWatch Alarm for canary rollback
  OrdersFunctionErrorAlarm:
    Type: AWS::CloudWatch::Alarm
    Properties:
      AlarmName: OrdersFunctionErrors
      MetricName: Errors
      Namespace: AWS/Lambda
      Statistic: Sum
      Period: 60
      EvaluationPeriods: 1
      Threshold: 5
      ComparisonOperator: GreaterThanThreshold
      Dimensions:
        - Name: FunctionName
          Value: !Ref OrdersFunction

Outputs:
  ApiUrl:
    Value: !Sub "https://${ServerlessRestApi}.execute-api.${AWS::Region}.amazonaws.com/Prod/"

☕ Java Lambda Handler — S3 Event

public class ImageProcessor implements RequestHandler<S3Event, Void> {

    private static final S3Client s3 = S3Client.create();

    @Override
    public Void handleRequest(S3Event event, Context context) {
        for (S3EventNotification.S3EventNotificationRecord record : event.getRecords()) {
            String bucket = record.getS3().getBucket().getName();
            // URL-decode the key (S3 encodes spaces as +)
            String key = record.getS3().getObject().getUrlDecodedKey();
            long size = record.getS3().getObject().getSizeAsLong();

            context.getLogger().log("Processing: s3://" + bucket + "/" + key + " (" + size + " bytes)");

            // Download original image
            ResponseBytes<GetObjectResponse> original = s3.getObjectAsBytes(r -> r
                .bucket(bucket).key(key));

            // Process (resize, thumbnail, etc.)
            byte[] thumbnail = createThumbnail(original.asByteArray(), 150, 150);

            // Upload thumbnail to different bucket/prefix
            String thumbnailKey = "thumbnails/" + key;
            s3.putObject(r -> r
                .bucket("thumbnails-bucket")
                .key(thumbnailKey)
                .contentType("image/jpeg"),
                RequestBody.fromBytes(thumbnail));

            context.getLogger().log("Thumbnail created: " + thumbnailKey);
        }
        return null;
    }
}
S3 → Lambda is ASYNC invocation. DLQ goes on the Lambda function (not S3). Lambda retries 3 times (0min, 1min, 2min) before sending to DLQ.
Two writes to same non-versioned object simultaneously may only deliver one event. Solution: enable versioning on the S3 bucket.

🎓 Exam Insights

  • S3 → Lambda = async invocation. 3 retries. DLQ on Lambda function.
  • S3 event notifications can also go to SQS or SNS (not just Lambda)
  • Object key filter: prefix (folder path) + suffix (file extension)
  • Enable S3 versioning to guarantee one event per write
  • S3 Object Lambda: transform data on-the-fly when reading (not on upload)

☕ Java — Human Approval Callback (WaitForTaskToken)

// Lambda that sends approval email and stores task token
public class SendApprovalEmail implements RequestHandler<Map<String, Object>, Void> {

    private static final SfnClient sfn = SfnClient.create();
    private static final DynamoDbClient ddb = DynamoDbClient.create();

    @Override
    public Void handleRequest(Map<String, Object> event, Context context) {
        String taskToken = (String) event.get("taskToken");
        String orderId = (String) event.get("orderId");

        // Store task token in DynamoDB so manager can retrieve it
        ddb.putItem(r -> r.tableName("PendingApprovals").item(Map.of(
            "orderId", AttributeValue.fromS(orderId),
            "taskToken", AttributeValue.fromS(taskToken),
            "expiresAt", AttributeValue.fromN(
                String.valueOf(Instant.now().plusSeconds(86400).getEpochSecond()))
        )));

        // Send email to manager with approve/reject links
        sendApprovalEmail(orderId, taskToken);
        return null;
    }
}

// Manager clicks "Approve" → this endpoint is called
@PostMapping("/orders/{orderId}/approve")
public ResponseEntity<Void> approveOrder(@PathVariable String orderId) {
    // Retrieve task token from DynamoDB
    String taskToken = getTaskToken(orderId);

    // Send success callback to Step Functions — workflow resumes!
    sfn.sendTaskSuccess(r -> r
        .taskToken(taskToken)
        .output("{\"approved\": true, \"approvedBy\": \"manager@company.com\"}"));

    return ResponseEntity.ok().build();
}

// Manager clicks "Reject"
@PostMapping("/orders/{orderId}/reject")
public ResponseEntity<Void> rejectOrder(@PathVariable String orderId, @RequestBody String reason) {
    String taskToken = getTaskToken(orderId);

    sfn.sendTaskFailure(r -> r
        .taskToken(taskToken)
        .error("OrderRejected")
        .cause(reason));

    return ResponseEntity.ok().build();
}

🎓 Exam Insights — Step Functions

  • Standard: max 1 year, exactly-once, 2000 starts/s — auditable, long-running
  • Express: max 5 min, at-least-once, 100K starts/s — high-volume IoT/streaming
  • Retry: exponential backoff with IntervalSeconds × BackoffRate^attempt
  • Catch: route to error state on specific error types
  • Parallel state: branches run concurrently (async); Task state: synchronous
  • WaitForTaskToken: pause workflow until external system calls SendTaskSuccess/Failure
  • Choice state: conditional branching (no retry/catch)
  • States.ALL: catch-all error handler

☕ Java — Generate CloudFront Signed URL

// Generate signed URL in Java
public String generateSignedUrl(String resourcePath, int expiryMinutes) throws Exception {
    Instant expiry = Instant.now().plusSeconds(expiryMinutes * 60L);

    // Load private key
    PrivateKey privateKey = loadPrivateKey("private_key.pem");

    SignedUrl signedUrl = CloudFrontUtilities.getSignedUrlWithCannedPolicy(
        SignRequest.builder()
            .resourceUrl("https://d1234.cloudfront.net/" + resourcePath)
            .keyPairId("APKA1234567890")  // CloudFront key pair ID
            .privateKey(privateKey)
            .expirationDate(expiry)
            .build());

    return signedUrl.url();
}

🎓 Exam Insights — CloudFront

  • OAI/OAC: S3 bucket only accessible via CloudFront — direct S3 URL = 403
  • Viewer Protocol Policy: "Redirect HTTP to HTTPS" or "HTTPS only" to force SSL
  • Signed URLs: per-file access control. Signed Cookies: multiple files at once
  • Cache invalidation: costs money. Use versioned file names instead (best practice)
  • Lambda@Edge: run Lambda at edge locations for viewer/origin request/response modification
  • CloudFront-Viewer-Country header: geo-restriction and routing
  • Price classes: limit edge locations to reduce cost

        // Use token as password — no hardcoded credentials!
        HikariConfig config = new HikariConfig();

        config.setUsername("lambda_user");
        config.setPassword(authToken);  // IAM token as password
        config.setMaximumPoolSize(5);
        config.addDataSourceProperty("sslMode", "REQUIRED");

        return new HikariDataSource(config);

☕ Java Lambda — SDK Clients (Correct Pattern)

/**
 * CORRECT: Static initialization = runs once on cold start, reused on warm invocations
 * WRONG: Creating clients inside handleRequest() = new client on every invocation (slow + wasteful)
 */
public class OrderProcessorLambda implements RequestHandler<APIGatewayProxyRequestEvent, APIGatewayProxyResponseEvent> {

    // ✅ Static = initialized once, reused across warm invocations
    private static final S3Client s3Client;
    private static final DynamoDbClient dynamoDbClient;
    private static final ObjectMapper objectMapper = new ObjectMapper();

    static {
        // In Lambda, DefaultCredentialsProvider automatically uses the execution role
        // No need to configure credentials — Lambda injects them via environment variables:
        // AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY, AWS_SESSION_TOKEN
        Region region = Region.of(System.getenv("AWS_REGION"));

        s3Client = S3Client.builder()
            .region(region)
            .credentialsProvider(DefaultCredentialsProvider.create())
            .build();

        dynamoDbClient = DynamoDbClient.builder()
            .region(region)
            .credentialsProvider(DefaultCredentialsProvider.create())
            .build();
    }

    @Override
    public APIGatewayProxyResponseEvent handleRequest(
            APIGatewayProxyRequestEvent event, Context context) {

        try {
            OrderRequest order = objectMapper.readValue(event.getBody(), OrderRequest.class);

            // 1. Save to DynamoDB
            Map<String, AttributeValue> item = Map.of(
                "orderId", AttributeValue.fromS(context.getAwsRequestId()),
                "customerId", AttributeValue.fromS(order.getCustomerId()),
                "amount", AttributeValue.fromN(order.getAmount().toString()),
                "status", AttributeValue.fromS("PENDING"),
                "createdAt", AttributeValue.fromS(Instant.now().toString())
            );
            dynamoDbClient.putItem(r -> r
                .tableName(System.getenv("TABLE_NAME"))
                .item(item));

            // 2. Upload order receipt to S3
            String receiptKey = "receipts/" + context.getAwsRequestId() + ".json";
            s3Client.putObject(r -> r
                .bucket(System.getenv("BUCKET_NAME"))
                .key(receiptKey)
                .contentType("application/json"),
                RequestBody.fromString(objectMapper.writeValueAsString(order)));

            return new APIGatewayProxyResponseEvent()
                .withStatusCode(201)
                .withBody("{\"orderId\":\"" + context.getAwsRequestId() + "\"}");

        } catch (Exception e) {
            context.getLogger().log("ERROR: " + e.getMessage());
            return new APIGatewayProxyResponseEvent()
                .withStatusCode(500)
                .withBody("{\"error\":\"Internal server error\"}");
        }
    }
}

☕ STS — Assume Cross-Account Role in Lambda

// Lambda in Account A needs to access S3 in Account B
public class CrossAccountLambda implements RequestHandler<Map<String, Object>, String> {

    private static final StsClient stsClient = StsClient.create();

    @Override
    public String handleRequest(Map<String, Object> event, Context context) {
        // Step 1: Assume role in Account B
        AssumeRoleResponse response = stsClient.assumeRole(r -> r
            .roleArn("arn:aws:iam::ACCOUNT_B:role/CrossAccountS3Role")
            .roleSessionName("lambda-session-" + context.getAwsRequestId())
            .durationSeconds(900));  // 15 minutes

        Credentials tempCreds = response.credentials();

        // Step 2: Create S3 client with temporary credentials
        S3Client crossAccountS3 = S3Client.builder()
            .region(Region.US_EAST_1)
            .credentialsProvider(StaticCredentialsProvider.create(
                AwsSessionCredentials.create(
                    tempCreds.accessKeyId(),
                    tempCreds.secretAccessKey(),
                    tempCreds.sessionToken()  // REQUIRED for temp credentials
                )))
            .build();

        // Step 3: Access Account B's S3
        ListObjectsV2Response objects = crossAccountS3.listObjectsV2(r -> r
            .bucket("account-b-bucket"));

        return "Found " + objects.keyCount() + " objects";
    }
}
Session token is REQUIRED when using temporary credentials from STS AssumeRole. Omitting it causes authentication failures.

🎓 Exam Insights — SDK & Credentials

  • Lambda credential chain: env vars (AWS_ACCESS_KEY_ID etc.) → IAM execution role
  • Initialize SDK clients in static blocks — reused across warm invocations
  • Cross-account: use STS AssumeRole — never share credentials
  • Temp credentials from STS: always include session token
  • Default region if not set: us-east-1
  • SDK exponential backoff: built-in for 429 (throttling) errors

@Service
public class MetricsPublisher {

    private static final CloudWatchClient cloudWatch = CloudWatchClient.create();

    // Call this after processing each order
    public void recordOrderMetrics(String customerId, double processingTimeMs, boolean success) {
        List<MetricDatum> metrics = List.of(
            MetricDatum.builder()
                .metricName("OrderProcessingTime")
                .value(processingTimeMs)
                .unit(StandardUnit.MILLISECONDS)
                .timestamp(Instant.now())
                .dimensions(
                    Dimension.builder().name("Environment").value("prod").build(),
                    Dimension.builder().name("Region").value("us-east-1").build()
                )
                .build(),
            MetricDatum.builder()
                .metricName(success ? "OrderSuccess" : "OrderFailure")
                .value(1.0)
                .unit(StandardUnit.COUNT)
                .timestamp(Instant.now())
                .build()
        );

        cloudWatch.putMetricData(r -> r
            .namespace("MyApp/Orders")
            .metricData(metrics));
    }
}

🎓 Exam Insights — CloudWatch

  • EC2 default: CPU, Network, Disk I/O — every 5 min. Detailed monitoring = 1 min (costs extra)
  • Memory, swap = NOT default — install CloudWatch Agent or push via PutMetricData
  • High-resolution custom metrics: up to 1 second (StorageResolution parameter)
  • Alarm states: OK, ALARM, INSUFFICIENT_DATA
  • Logs Insights: interactive query language for log analysis. Not real-time.
  • Log group encryption: KMS at group level. Cannot specify KMS key via CLI create-log-group.
  • CloudWatch Events / EventBridge: react to AWS service events or schedule cron jobs
